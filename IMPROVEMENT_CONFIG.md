# Verbeterconfiguratie

Dit bestand stuurt de dagelijkse automatische verbeteringen aan het project.

## Prioriteiten

1. **Code kwaliteit** - Consistente stijl, geen duplicatie, duidelijke naamgeving
2. **Tests** - Testdekking verhogen, edge cases testen
3. **Documentatie** - README bijhouden, inline comments waar nodig
4. **Security** - Geen hardcoded secrets, input validatie, dependency updates
5. **Performance** - Onnodige operaties vermijden, efficiënte algoritmes

## Regels voor wijzigingen

- Maximaal **1-2 bestanden** per verbetering
- Elke wijziging moet **backward compatible** zijn
- **Tests** moeten blijven slagen na elke wijziging
- Commit messages in het **Nederlands**
- Geen cosmetische wijzigingen zonder functionele waarde

## Gebieden om te negeren

- Gegenereerde bestanden (build output, lock files)
- Vendor/third-party code
- Configuratie die omgevingsspecifiek is

> Pas dit aan om de dagelijkse verbeteringen te sturen naar wat nu het
> belangrijkst is voor het project.

- [ ] Basisstructuur van het project opzetten
- [ ] README.md schrijven
- [ ] Eerste tests toevoegen
